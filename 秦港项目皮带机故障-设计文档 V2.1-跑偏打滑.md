AI分析-皮带机跑偏故障
------------

当运营运维人员在AI分析页面点击皮带机跑偏故障列操作按钮“深度分析”时候，根据当前故障ID、设备名称进行一个深度分析，具体流程如下：

<img src="{PUBLIC_ACCESS_DOMAIN}/duowen_searcher/preview/image/markdown/78daca15c9d640058bb0b111da0a206f/images/xDUfV2HYGbg8CgrR.png"/>

### 查询告警详情，定位故障位置

|  |  |  |  |
| --- | --- | --- | --- |
| qgznyw.gis_plc_fault | | | |
| 字段名称 | 字段类型 | 字段释义 | 示例 |
| ID | NUMBER | 主键 | 1105736 |
| F_EQP_NAME | VARCHAR2 | 设备名称 | BM11 |
| F_START_DATE | DATE | 故障时间 | 2025/6/1 12:56:00 |
| F_REASON | VARCHAR2 | 故障内容描述 | 跑偏报警1,2号异常 |
| FNAME | VARCHAR2 | 数据采集标签描述 | 跑偏报警1,2号 |
| F_TYPE | NUMBER | 故障类型（1:故障 2：报警 5:转接塔故障） | 2 |
| FCODE | VARCHAR2 | 数据采集标签 | [TOPIC_SS1]BM11_XX.PP1_Alarm |
| FID | NUMBER | 数据采集ID | 132 |
| STATUS | NUMBER | 状态(0:未处理 1:已处理) | 1 |
| FNOTES | VARCHAR2 | 转接塔名称 |  |

* 查询条件：AI分析页面点击深度分析或者弹窗页面，告警ID或设备名称+故障时间
* 位置判断逻辑：根据FNAME字段内容判断跑偏故障位置信息，如：跑偏1、2号，代表尾部跑偏开关1号和2号位置发出了报警，以此类推3、4号为中间位置，详见《皮带参数表》**需要提供跑偏开关、打滑开关数量、编号、位置（或者位置逻辑）**
* 呈现内容：故障位置、设备名称、故障时间、故障内容描述、故障类型、处理状态（未处理、已处理）、转接塔名称；

### 查询皮带机故障位置视频

* 提供多个视频链接，高亮显示
* 视频获取逻辑：如果无法获取故障位置摄像头，可提供皮带沿线摄像头，根据视频表gis_video表中ADDRESS字段模糊筛选关键字“皮带沿线”+FEQUCODE模糊筛选设备名称获取摄像头code码，多个code码提供多个视频链接；
* **人工判断”是/否“真实跑偏**

### 查询是否湿黏煤

* 查询条件：设备编号+故障时间（小时粒度）
* 处理逻辑：查询《流程作业表》，设备名称（BD10）+故障时间（2025-11-14 13）筛选流程作业表中煤种编号，根据煤种编号到《煤种基础表》查询煤种信息；获取普通煤、湿煤信息；
* 具体呈现内容：煤种信息（普通煤、湿煤），根据煤种信息判断是否易粘连，导致皮带跑偏；
